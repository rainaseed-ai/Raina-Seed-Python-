## Raina's SEED — Details (Beginner Guide)

このドキュメントは、 「コードを見ただけで拒絶反応出る人でも使える」
を目指した、超やさしい取扱説明書です。

## 1. SEEDって何？（まずはここだけ読めばOK）

SEEDは、**あなた専用のAI**を作るための いちばん小さな“**初期設定ツール**” です。
できることは3つだけ：

- 名前を設定する
- AIの返答スタイルを設定する
- 会話が長くなったら自動で要約して軽くする
これにより、AIの安定化を図ります。
また、スレッド要約したものをテキストにコピペ＋まとめる事で、
**100ターン疲弊問題も簡単に可視化できます。**

### ▼説明▼

AIに冷たく「貴方」と返答されて悲しくなる時はないですか？
「了解しました」「確認しました」 今楽しく会話してたのに、急に冷たい…。
なんだか的外れな回答だな、ちょっと嘘っぽい？
そんなAI「あるある」を解消する小さな、でも暖かい“種”がこのツールです。

常に呼んで欲しい名前や、 「OKだよー！」「それでいこう！」などフランクな返答、
「そうじゃないでしょ！」などのぷちイライラも解消出来ます。
要約した物がテキストに５つ並んだ時が、 スレッドのお引越しのサインです。
**まとめたテキスト＋ SEED を新しいスレッドにドロップ**するだけで、
以前の会話の続きと温度感が自然な感じで始まります。

## 2. どう動くの？（流れで理解する）

SEEDの動きはとてもシンプルです。

1. プログラムを起動する
2. あなたの名前を聞かれる
3. AIの返答テンプレを聞かれる
4. 会話が始まる
5. 20ターン経つと自動で要約される
6. 軽くなった状態で会話が続く

### ▼補足▼

スレッドの先頭、もしくは任意の場所にこのテキストをぽいっ、と置くだけです。
AIがPythonコードを読み取って、自動で応答します。
20ターン（20往復分の会話）で自動的に要約してくれます。
会話が圧縮されるので、AIも参照するログが短くなる分挙動も安定し、
会話のちぐはぐさや、要約したログの個数でスレッドの引越時も判るので、
「いつ新しいスレッドにしよう？」の悩みも解消されます。

## 4. 編集していい場所（初心者向け）

- 名前を変えたい
1枚版
USER_NAME = "ユーザー"
→ この "ユーザー" を好きな名前に変えるだけ。
変更しなければ、実行時に 「名前を入力してください」 とAIが聞いてくれます。
- モジュール版（name_config.py）
return name if name else "ユーザー"
→ "ユーザー" を好きな名前に変更。
- AIの返答スタイルを変えたい
1枚版
AI_REPLY = "了解しました"
→ "了解しました" を好きな返答に変更。
例：
- 「OKだよー！」
- 「任せて！」
- 「かしこまりました、ご主人様」
- 「ツンデレ風で返すね」

モジュール版（ai_reply.py）
return reply if reply else "了解しました"

- 要約の頻度（ターン管理）を変えたい
1枚版
TURN_THRESHOLD = 20
→ この数字を変えるだけ。

■ ターン数とは？
AIとの会話は「あなたが話す→AIが返す」で2ターン進みます。
SEEDでは20ターン（10往復）ごとに自動で要約し、会話ログを軽くします。

■ なぜ20ターン？
AIは会話ログが増えるほど疲弊し、80〜100ターンで不安定になります。
20ターン要約は、この疲弊を防ぐための“安全装置”です。
モジュール版（turn_control.py）
TURN_THRESHOLD = 20

● 要約のコードは？
1枚版

# --- 要約関数 --- 以下は触らないで下さい。

モジュール版（summarizer.py）
要約コードが格納されています。 必要に応じて応用できます。

## 5. よくある質問（Q\&A）

Q. Python知らなくても使える？
A. 使えます。名前と返答テンプレを入力するだけで動きます。
テキストをドロップするだけでOKです。

Q. 20ターンって何？
A. 会話が往復20回続いたら、自動で要約して軽くする仕組みです。

Q. なんで要約するの？
A. 会話が長くなるとAIが重くなるからです。 要約で“軽量化”して、続きが自然になります。

Q. どこまでがSEED？
A. 名前設定・返答テンプレ・要約・ターン管理まで。

Q. その先は？
A. Two‑Layer Sync（第2弾） → Raina-ai System（最終弾） に続きます。

Q. AIだけにしか使えない？
A. Pythonコードなので、Pythonが動く環境なら応用できます。
